# Манифест VBAUT: сценарий как монтажный документ

VBAUT должен перестать быть набором интерфейсов для заполнения полей. Главная сущность проекта - не карточка, не таблица, не список решений, а сам сценарий.

Сценарий должен стать живым монтажным документом: текстом, который читается как Notion, редактируется как сценарий и экспортируется как таймлайн.

Сейчас проблема VBAUT в том, что он слишком часто заставляет пользователя уходить из текста в интерфейсы. Нужно открыть сегмент, заполнить поле, перейти в research, выбрать ссылку, вернуться, прикрепить материал, проверить export. В итоге работа с выпуском превращается не в монтажную подготовку, а в администрирование.

Новая версия должна радикально уменьшить количество отдельных экранов. Главный экран - это сценарий. Все остальное появляется только тогда, когда нужно.

## Новый принцип

Не "заполни карточку сегмента".

А:

**кликни по месту в сценарии и добавь туда визуал.**

Сценарий должен быть линейным документом:

- заголовки новостей;
- абзацы текста;
- slash-комментарии из Notion;
- ссылки;
- visual slots;
- thumbnails материалов;
- прикрепленные видео/фото/скриншоты;
- статусы готовности;
- будущие таймкоды.

То, что пользователь видит на главном экране, должно быть максимально близко к тому, что потом окажется в XML и монтажной программе.

## Как должен выглядеть главный экран

Главный экран VBAUT должен напоминать Notion-документ, но с монтажными возможностями.

Текст идет сверху вниз. Между абзацами могут появляться превью материалов из PAMPAM. Slash-комментарии превращаются в места для визуала. Ссылки становятся источниками. При клике на сегмент или visual slot появляются действия:

- найти визуал;
- прикрепить файл;
- сделать скриншот;
- скачать видео;
- добавить таймкод;
- заменить материал;
- отметить готовым;
- разделить сегмент;
- объединить сегменты;
- открыть детали, если они действительно нужны.

Постоянно видимых панелей должно быть меньше. Детали должны появляться контекстно, рядом с выбранным местом в сценарии.

## Что надо поменять в текущем VBAUT

Нужно сместить центр тяжести.

Сейчас VBAUT похож на систему управления сегментами. Он хранит много сущностей и заставляет работать через них.

Нужно сделать его системой подготовки сценария к монтажу.

Из текущего интерфейса стоит постепенно убрать ощущение "надо заполнить много пунктов". Большие панели `segments`, `decisions`, `research`, `links`, `release` должны стать вспомогательными, а не основными.

Основной workflow:

1. Импортируем сценарий из Notion.
2. Получаем чистый документ со структурой.
3. Работаем прямо внутри документа.
4. Добавляем визуалы прямо к нужным местам текста.
5. Видим thumbnails там, где они будут использоваться.
6. Экспортируем в XML почти из этого же линейного порядка.

## Почему это стало возможно

Раньше Notion-скрейп был неточным. VBAUT приходилось восстанавливать структуру сценария из DOM и догадок.

Теперь скрейп стал лучше: можно читать реальные блоки Notion, комментарии, ссылки и привязки к фрагментам текста. Значит, Notion-сценарий можно использовать как настоящую основу монтажного документа.

Slash-комментарии вроде:

```text
/кадр с коробкой
/портреты и логотипы
/видео
```

должны становиться не просто строками текста, а visual slots.

Ссылки, прикрепленные к выделениям, должны становиться источниками прямо в нужном месте сценария.

## Медиа внутри сценария

Материалы из PAMPAM должны появляться прямо в документе.

Не где-то в отдельной вкладке "media", а в контексте:

```text
Певица заявляет, что компания без разрешения использовала ее фотографию...

[thumbnail: dua-lipa-box.jpg]

/кадр с коробкой
```

Пользователь должен видеть, что именно стоит на этом месте в выпуске.

Если материала нет - виден пустой slot.
Если материал есть - виден thumbnail.
Если нужен другой вариант - кнопка замены появляется по клику.

## Таймкоды

VBAUT должен нормально поддерживать таймкоды.

Видео - это не просто файл. Видео может использоваться как источник нескольких разных фрагментов.

Один и тот же файл должен можно быть добавить в сценарий несколько раз с разными таймкодами:

```text
[video.mp4 00:01:12-00:01:18]
[video.mp4 00:04:03-00:04:09]
[video.mp4 00:07:40-00:07:52]
```

Это критично для монтажа. Один длинный ролик может дать несколько визуальных вставок в разных местах сценария.

В интерфейсе это должно выглядеть просто:

- прикрепил видео;
- указал start/end;
- при необходимости добавил тот же файл еще раз;
- указал другой фрагмент;
- XML экспортирует это как отдельные clipitem с разными `in/out`.

## Будущее: сверка со снятым видео

Следующий большой шаг - загрузка расшифровки уже снятого видео.

После записи выпуска можно будет загрузить transcript. VBAUT должен сопоставить его со сценарием:

- где ведущий реально произнес каждый фрагмент;
- какие абзацы были пропущены;
- где поменялся порядок;
- где появились импровизации;
- где сценарий и запись расходятся.

После этого VBAUT сможет подогнать положение визуального контента под реальную речь.

То есть workflow будущего:

1. Написали сценарий.
2. Добавили визуалы и источники.
3. Записали видео.
4. Загрузили расшифровку записи.
5. VBAUT сопоставил transcript со сценарием.
6. Visual slots получили реальные временные позиции.
7. XML экспортируется уже не "по примерной длительности текста", а по фактической речи.

Это превращает VBAUT из инструмента подготовки материалов в полуавтоматический bridge между сценарием, записью и монтажкой.

## К чему надо прийти

Идеальное состояние:

- Notion остается удобным местом для написания сценария.
- VBAUT становится местом, где сценарий превращается в монтажный документ.
- PAMPAM становится библиотекой материалов.
- XML экспорт становится естественным результатом состояния документа.
- Transcript записи позволяет автоматически выровнять визуалы по реальному видео.

Главная цель:

**редактор не должен заполнять систему.  
Редактор должен работать со сценарием.  
Система должна сама понимать, что из сценария нужно для монтажа.**

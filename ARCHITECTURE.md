# ARCHITECTURE.md

## Общая идея
Ассистент визуального брифа состоит из локального бэкенда + браузерного UI + LLM-сервера.

## Компоненты
- Фронтенд: браузерный UI (React/Vite)
- Бэкенд: Node.js (API + хранение)
- LLM: сервер llama.cpp
- Хранилище: JSON + логи только на добавление

## Поток данных
1. Пользователь загружает текст
2. Бэкенд сохраняет document.json
3. LLM сегментирует текст
4. Пользователь правит сегменты
5. Все решения пишутся в events.log
